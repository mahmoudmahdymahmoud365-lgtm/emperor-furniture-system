# دليل الربط والتكامل المستقبلي - الامبراطور للأثاث

## الوضع الحالي

التطبيق يعمل حالياً بنظام تخزين محلي (localStorage + IndexedDB) مع طبقة بيانات موحدة (`src/data/store.ts`) تسهّل الانتقال لأي قاعدة بيانات مستقبلاً.

---

## 🔌 خيارات الربط المستقبلية

### 1. Lovable Cloud (الخيار الموصى به)

**المميزات:**
- قاعدة بيانات PostgreSQL جاهزة بدون إعداد
- نظام مصادقة مدمج (تسجيل دخول / خروج / نسيت كلمة المرور)
- تخزين ملفات (صور المنتجات، الوثائق)
- دوال سحابية (Edge Functions) لإرسال الإيميلات، الدفع الإلكتروني
- أمان على مستوى الصفوف (Row Level Security)

**خطوات التفعيل:**
1. اطلب من المساعد "فعّل Lovable Cloud"
2. سيتم إنشاء الجداول تلقائياً
3. تحديث طبقة البيانات للاتصال بالقاعدة
4. تفعيل المصادقة الحقيقية بدلاً من localStorage

**ما سيتغير:**
- `src/data/store.ts` → سيستخدم Supabase Client بدلاً من localStorage
- المصادقة → ستنتقل من كلمات مرور محلية إلى نظام Supabase Auth
- الصور → ستُخزن في Supabase Storage بدلاً من IndexedDB

---

### 2. الباك إند الموجود (Express + PostgreSQL)

**المتطلبات:**
- تشغيل `backend/` على سيرفر خارجي
- ضبط `API_BASE_URL` في `src/data/apiClient.ts`
- تفعيل `src/data/store.api.ts` كبديل لـ `store.ts`

**خطوات الربط:**
1. ارفع مجلد `backend/` على سيرفر (Railway, Render, VPS)
2. أنشئ قاعدة PostgreSQL وشغّل `docs/postgresql_schema.sql`
3. اضبط متغيرات البيئة في `.env`
4. غيّر الـ import في التطبيق ليستخدم `store.api.ts`

---

### 3. نسخة سطح المكتب (Electron + SQLite)

**المتطلبات:**
- اتبع `ELECTRON_SETUP.md`
- قاعدة بيانات SQLite محلية تعمل بدون إنترنت
- يستخدم `src/data/store.electron.ts`

---

## 🔑 تكاملات خارجية مقترحة

### بوابة الدفع (Stripe)
- تتطلب Edge Function في Lovable Cloud
- المفتاح السري يُخزن في Cloud Secrets
- المفتاح العام (Publishable) آمن في الكود

### إرسال الإيميلات (Resend / SendGrid)
- تتطلب Edge Function
- API Key يُخزن في Cloud Secrets
- مفيد لإرسال الفواتير والتنبيهات للعملاء

### واتساب بيزنس API
- تتطلب Edge Function + Webhook
- للإشعارات التلقائية وتأكيد الطلبات

### تسجيل الدخول بـ Google / Apple
- مدعوم مباشرة في Lovable Cloud
- بدون إعداد إضافي

## 🏗️ هيكل الكود الجاهز للترقية

```
src/data/
├── types.ts          # أنواع البيانات (لا تتغير)
├── hooks.ts          # React hooks (لا تتغير)
├── store.ts          # التخزين المحلي الحالي
├── store.api.ts      # جاهز للربط بـ REST API
├── store.electron.ts # جاهز لـ Electron/SQLite
├── apiClient.ts      # HTTP client للباك إند
└── imageStore.ts     # IndexedDB للصور
```

**الميزة الأساسية:** تغيير مصدر البيانات يتطلب فقط تبديل الـ import في الملفات، بدون تعديل أي مكوّن UI.

---

## ⚠️ ملاحظات أمنية

- **لا تخزن مفاتيح API السرية في الكود** — استخدم Cloud Secrets
- **المفاتيح العامة (Publishable)** آمنة في الكود
- **لا يوجد ملف `.env`** في Lovable — استخدم Cloud Secrets بدلاً منه
- **عند الانتقال لقاعدة بيانات حقيقية:** فعّل RLS وأضف تحقق من الصلاحيات على مستوى السيرفر
